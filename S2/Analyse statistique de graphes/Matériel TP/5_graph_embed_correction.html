# calcul de la similiarit gaussienne entre deux points dans R^p
def simi_exp(x1, x2, sigma = 1):
    return np.exp(-np.sum((x1 - x2)**2) /(2 * sigma**2))

# test de la fonction 
#simi_exp(np.array([1,2]),np.array([3,4]))

# construction de la matrice de similarit gaussienne d'un jeu de donnes
def similarity(df):
    n =len(df)
    S=np.zeros(shape=[n,n])
    for i in range(n-1):
        for j in range(i+1, n):
            S[i,j] = simi_exp(df[i, ], df[j, ])
            
    S = S+S.transpose()
    return S

Le plongement ne nous a pas permis de sparer les noeuds.

Pourquoi ?

C'est trs simple : DeepWalk ne prend pas en compte les valeurs d'un graphe valu (il ne considre que sa version binarise). Or, on lui a donn un graphe de similarit exponentielle sans faire de seuillage : donc le graphe est complet. Il ne contient aucune information de communaut. partir d'un noeud, tous les autres noeuds du graphe ont la mme proba uniforme d'tre visit.

Il fallait donc faire un graphe avec un seuil de similarit. On recommence.

Node2vec¶

On rappelle que Node2vec est une version amliore de DeepWalk, dans laquelle la marche alatoire est biaise. La page officielle de Node2Vec est ici https://github.com/eliorc/node2vec On va suivre l'exemple en dmo.

On va ensuite tester la mthode sur le graphe de similarit G obtenu ci-dessus.

Les options de Node2Vec sont similaires celles de DeepWalk

  • dimensions: Embedding dimensions (default: 128)
  • walk_length: Number of nodes in each walk (default: 80)
  • num_walks: Number of walks per node (default: 10)
  • p: Return hyper parameter (default: 1)
  • q: Inout parameter (default: 1)

Si vous laissez les valeurs de p,q par dfaut (= 1), vous retombez sur la mthode DeepWalk quand vous traitez un graphe binaire, donc a n'a pas d'intrt.

Par contre, dans le cas valu, Node2Vec va prendre en compte les valeurs des artes donc mme avec $p=q=1$, on ne devrait pas avoir le mme rsultat que DeepWalk.

C'est ce qu'on va vrifier avec le graphe de similarit complet ci-dessus.

Notez que contrairement DeepWalk :

  • les donnes se passent dans le modle (parce que les probas de la marche alatoire dpendent des voisinages ; quand on rflchit a, on comprend pourquoi dans l'implmentation de Node2Vec, c'est pas plus compliqu de rajouter les poids des artes, ce qui a donc t fait. d'un point de vue architecture logicielle, c'est quand mme nettement plus compliqu que DeepWalk).
    • la taille de la fentre window n'est pas un paramtre du modle mais de la fonction d'ajustement aux donnes fit
